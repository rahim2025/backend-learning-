# What Node.js is Commonly Used For

Node.js is commonly used for backend and server-side development.

## Common Uses

| Use case | Example |
| --- | --- |
| REST APIs | User API, product API, auth API |
| Real-time apps | Chat, notifications, live dashboards |
| Microservices | Small backend services |
| Command-line tools | Scripts and developer tools |
| File processing | Uploads, logs, reports |
| Streaming | Video, audio, data streams |
| Backend for frontend apps | API server for React/Vue/Angular |

## Why Node.js is Good for I/O

Node.js is good for I/O-heavy applications because it uses non-blocking asynchronous operations.

While Node.js waits for a database, file, or network response, it can continue handling other work.

## Interview Answer

Node.js is commonly used for REST APIs, real-time apps, microservices, command-line tools, file processing, streams, and I/O-heavy backend systems.
